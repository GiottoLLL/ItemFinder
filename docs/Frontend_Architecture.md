# 物归处 (ItemFinder) 前端架构与开发规范文档

## 1. 页面解耦与路由栈设计 (React Navigation)

为了保证前端代码的高可维护性与复用性，我们将页面严格划分为**底部导航栈 (Tab Stack)** 和 **普通堆栈 (Native Stack)**。

### 1.1 底部导航栈 (Tab Stack)
这部分是 App 的核心四大主页，带有统一的底部导航栏：
- **`HomeDashboard` (首页)**：仪表盘设计，展示物品总数和常用物品快捷入口
- **`ItemEntrySelection` (录入页)**：多渠道入口页，提供拍照录入、扫码识别、票据导入和手动录入的选项
- **`ItemSearch` (查找物品页)**：提供手动关键字查找或 AI 智能语音/文字查找
- **`InventoryList` (物品列表页)**：卡片列表形式展示用户的全局库存，支持关键字搜索和储物空间维度筛选

### 1.2 普通堆栈 (Native Stack)
从主页衍生出的功能页或操作页，隐藏底部导航栏：
- **`ItemEntryForm` (录入表单页)**：实际输入物品属性（数量、存放位置等）的页面
- **`ItemDetail` (物品详情页)**：展示物品详细存放位置，若为消耗品则展示当前余量
- **`StorageSpaceList` (储物空间列表页)**：用户自定义管理的储物空间列表
- **`AuthLogin` (登录页)**：支持账号密码登录、手机号验证码登录
- **`AuthRegister` (注册页)**：新用户注册链路

## 2. 全局公共组件抽象 (Components)

这些组件在多处被复用或具备全局展示属性，需在 `src/components` 中独立封装：
- **`BottomTabBar` (底部导航栏)**：承载 Tab Stack 的切换逻辑
- **`ButlerFAB` (悬浮管家按钮)**：全局悬浮于右下角，点击后唤起交互
- **`ButlerChatMask` (管家交互遮罩层)**：带遮罩背景，包含底部语音/文字输入框
- **`ButlerResponseCard` (管家应答卡片)**：在顶部弹出的卡片，用于展示 AI 查找结果或快捷操作确认
- **`AddStorageModal` (添加存储空间弹窗)**：带遮罩层，在录入表单页和储物空间列表页复用
- **`InventoryWarningModal` (库存预警弹窗)**：App 启动时或特定生命周期触发的全局提示

## 3. 静态页面开发流程与 Figma 协同

1. **Figma 数据提取**：
   - 开发者提供 Figma 的 `fileKey`（设计稿 URL 中的标识）
   - 提供对应页面的 `nodeId`（选中具体 Frame 后的节点标识，如 `10:25`）
   - 智能体通过 MCP 工具读取精确的布局、颜色和图层数据。
2. **UI 还原与组件拆分**：
   - 优先完成静态组件的 Flex 布局与样式还原
   - 识别页面中的重复元素（如按钮、卡片、表单项）并提炼到公共组件库
3. **交互占位**：
   - 暂不接入真实数据库，使用空 Promise 或 `console.log` 对按钮点击、表单提交等事件进行占位。

## 4. 前端开发规范与注意事项 (严格执行)

- **包管理工具**：强制使用 `pnpm` (当前 Node 环境要求 18.20.8 及以上)
- **命名规范**：
  - 组件/页面文件：大驼峰命名 (`PascalCase`，如 `ItemDetail.jsx`)
  - 变量/函数/普通脚本：小驼峰命名 (`camelCase`)
- **代码风格与语法**：
  - **分号规范**：生成 JS、JSX 等前端脚本时，**严禁以分号 `;` 结尾**
  - **ES6+ 优先**：严格遵守 ES6+ 规范
  - **条件判断**：`if` 语句中，条件表达式必须用括号括起来，如 `if (condition) { ... }`
- **异步处理**：
  - 优先考虑使用 **`Promise`** 链式调用 (`.then().catch()`)，尽量**避免**使用 `async/await`
- **代码质量**：
  - 强调**语义化**，HTML/JSX 标签及变量命名需清晰表达业务意图
  - 复杂逻辑处必须增加中文注释，解释“为什么这么做”

## 5. 后续规划（致数据结构与数据库设计智能体）

前端页面结构已解耦完毕。下一个阶段需基于此文档和《MVP.md》设计本地 SQLite 的数据结构。
**核心需考虑的实体模型包括：**
- **物品 (Item)**：需包含名称、图片路径、所属位置ID、数量、是否为消耗品、录入时间等
- **储物空间 (StorageSpace)**：支持层级关系或平铺列表
- **预警规则 (WarningRule)**：用于支撑“库存预警弹窗”的 AI 消耗计算逻辑
