# Bug 处理与优化待办

## 文档说明

- 文档目标：整理当前阶段确认的 Bug、交互优化、功能迭代与智能管家相关预研项
- 适用阶段：当前 MVP 持续开发阶段
- 排期原则：
  - `P0`：直接影响主流程、核心闭环、基础导航与当前可用性
  - `P1`：高价值交互优化与下一步重点功能铺垫
  - `P2`：智能管家深水区能力、数据库与 AI 能力扩展

---

## P0

### 01. 首页统计卡跳转物品列表

- 问题：首页“物品总数”卡片点击后未跳转到物品列表页
- 目标：点击后进入物品列表页
- 优先级：`P0`

### 02. 常用物品跳转详情页

- 问题：首页“常用物品”区域点击名称后未跳转到物品详情页
- 目标：点击物品名称进入对应物品详情页
- 优先级：`P0`

### 03. 拍照录入主流程打通

- 问题：拍照录入当前仅为占位跳转
- 目标：
  - 点击后尝试调用相机拍照
  - 拍照成功后进入录入物品页
  - 保存物品信息时同步保存照片信息
  - 若当前数据库未完整支持图片存储，则补齐数据库设计
- 涉及范围：
  - 相机权限与拍照能力
  - 录入页图片展示与删除
  - 数据库图片字段或图片表设计
- 优先级：`P0`

### 04. 存放位置改造为可搜索下拉框

- 问题：当前存放位置选择方式不符合目标交互
- 目标：
  - 点击“选择存放位置”后展示下拉框
  - 下拉框第一项为搜索输入
  - 下拉框最后一项为“新增存储空间”
  - 中间为可上下滚动的存储空间列表
- 优先级：`P0`

### 05. 接入扫码识别

- 问题：扫码识别当前仅为占位
- 目标：
  - 接入条形码扫描依赖库
  - 打通扫码录入基础流程
- 优先级：`P0`

### 06. 录入页增加图片模块

- 问题：录入物品页缺少图片添加能力
- 目标：
  - 无图片时显示方形添加按钮
  - 有图片时显示图片列表
  - 每张图片右上角有删除按钮
- 优先级：`P0`

### 07. 去掉票据导入

- 问题：票据导入不再需要
- 目标：移除票据导入入口、相关文案和后续无用逻辑
- 优先级：`P0`

### 08. 新增批量导入入口

- 问题：票据导入移除后需要新的替代能力
- 目标：增加“批量导入”入口来代替票据导入
- 优先级：`P0`

### 09. 新建批量导入页

- 目标：
  - 按照 `docs/DESIGN.md` 的设计风格实现批量导入页
  - 点击批量导入后进入该页面
  - 批量导入以列表方式呈现
  - 支持“添加一项”和“删除一项”
  - 每项仅填写：物品名称、数量、单位、存放位置
- 优先级：`P0`

### 10. 继续录入按钮改为录入

- 问题：当前“继续录入”不符合预期行为
- 目标：
  - 按钮文案改为“录入”
  - 输入物品名称后点击按钮直接录入
  - 录入完成后自动清空输入框
- 优先级：`P0`

### 11. 搜索触发时机与搜索记录修正

- 问题：
  - 当前搜索不应实时触发
  - 搜索词未正确进入搜索记录
- 目标：
  - 仅在用户按 `Enter` 或点击搜索按钮后再执行搜索
  - 搜索词正确加入搜索记录
- 优先级：`P0`

### 12. 切换到录入页时回到顶部

- 问题：切换到录入页时未回到顶部
- 目标：进入录入页时自动滚动到页面顶部
- 优先级：`P0`

### 13. 底部栏切页时顶部标题联动

- 目标：底部栏切换页面时，顶部标题跟随变化
- 标题顺序：
  - 主页
  - 录入物品

- 优先级：`P0`

### 14. 侧边栏能力补齐

- 问题：左上角点击项需要打开侧边栏，但当前能力不完整
- 目标：
  - 若无侧边栏则补做
  - 侧边栏包含：
    - 存储空间：点击后跳转存储空间列表

  - 侧边栏触发的预警弹窗不提供“不再提示”选项
- 优先级：`P0`

### 15. 去掉当前关闭按钮

- 问题：当前页面上该关闭按钮不再需要
- 目标：移除该关闭按钮及无用交互

## P2

### 20. 智能管家能力建设

- 产品定位：
  - 类似知道所有物品可能在哪、还有多少的“妈妈型管家”
- 目标能力：
  - 查找物品
    - 示例：“我数据线收哪了？”
    - 返回多个可能位置，如手机数据线、Switch 数据线分别在哪
  - 智能录入
    - 示例：“我买了运动相机，放防潮箱了”
    - 若没有对应存储空间，可自动创建
  - 物品管理
    - 示例：“我牙膏还有多少”
    - 基于上次更新日期、历史信息、推测逻辑估算余量
  - 智能推测
    - 若用户反馈“在沙发底下找到了”
    - 记录结果
    - 下次查询同类物品时给出补充建议
- 需要同步推进的事项：
  - 将“智能管家”加入 `docs/MVP.md`
  - 预留 AI 对接工具类
  - 明确数据库是否需要新增：
    - 智能查询记录
    - 对话消息
    - 用户反馈结果
    - 位置推测经验
    - 用量推测相关数据
- 当前状态说明：
  - 该能力涉及产品定义、页面、数据库、服务层、AI 接口层与后续提示词设计
  - 建议拆阶段实施，而不是一次性全部完成
- 优先级：`P2`
