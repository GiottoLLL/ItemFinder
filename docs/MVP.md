### **MVP文档：物归处 (ItemFinder) - 个人物品管理App**
**版本：** 1.1 (重构版)
**日期：** 2026年3月  
**作者：** Giotto 
---
#### **1. 产品概述**
**产品名称：** 物归处 (ItemFinder)  
**核心价值：** 通过AI辅助的快速录入和简单的本地搜索，解决用户“找不到东西”的痛点，帮助个人高效管理物品存放位置。  
**目标用户：** 经常忘记物品存放位置的个人（如家庭主妇、学生、租房族）。  
**核心痛点：** 买了东西但忘记放在哪里；重复购买已拥有的物品；整理房间时找不到所需物品。  
---
#### **2. 核心指标 (Success Metrics)**
- **北极星指标：** 用户平均每周检索成功次数。
- **体验指标：** 从打开App到完成一件物品录入的平均时长（目标：<10秒）。
---
#### **3. 核心用户路径 (User Flow)**
- **快速录入闭环：** 打开App -> 点击拍照 -> AI自动识别物品和位置 -> 用户核对修改（可选） -> 点击保存 -> 完成。
- **查找闭环：** 打开App -> 搜索框输入关键词（如“充电线”） -> 列表展示匹配物品及位置 -> 找到物品。
---
#### **4. 核心功能 (MVP P0阶段)**
*注：MVP阶段砍掉账号登录，采用设备本地存储，开箱即用。*

##### **4.1 物品录入 (Item Entry)**
- **4.1.1 拍照智能录入 (主打功能)**  
  - **功能描述：** 用户拍照，AI识别物品名称（如“方便面”）和可能的位置背景。  
  - **技术实现：** 调用智谱GLM-4V-Flash图像识别API，返回结构化数据。  
  - **网络依赖：** 需联网。断网时提示“当前无网络，无法使用智能识别”，并自动降级为手动录入模式。
- **4.1.2 手动录入 (基础兜底 & 离线支持)**  
  - **功能描述：** 用户手动输入物品名称、数量和位置。  
  - **技术实现：** 纯本地表单交互。  

##### **4.2 位置与数据管理 (Data Management)**
- **功能描述：** 纯本地保存物品与位置的映射关系。支持用户手动修改或删除。  
- **技术实现：** SQLite 本地数据库。  

##### **4.3 搜索功能 (Search)**
- **模糊搜索：**  
  - **功能描述：** 根据物品名称或位置进行文本模糊匹配。  
  - **技术实现：** 基于 SQLite 的 `LIKE` 查询或简单的本地全文检索。  

---
#### **5. 技术栈**
| 类别       | 推荐工具                  | 理由                                  |
|------------|---------------------------|---------------------------------------|
| 前端       | React Native              | 跨平台、生态成熟、适合快速构建MVP       |
| 数据存储   | SQLite                    | 纯本地存储，无后端成本，保护隐私        |
| AI服务     | 智谱GLM-4V-Flash          | 免费、图像识别能力强，匹配MVP需求       |
| 状态管理   | Redux Toolkit             | 适合复杂状态管理                        |
| 导航       | React Navigation          | 简单易用、社区活跃                      |
| UI设计     | Google IDX / Stitch       | AI辅助设计，更贴合极简开发流          |
---
#### **6. 隐私与安全**
- **纯本地化：** 无账号体系，所有数据（含照片、位置）均保存在本地，不上传云端（仅拍照识别时将图片发给大模型API，不留存）。
- **隐私政策：** 首次打开需同意隐私声明。
---
#### **7. 发布计划 (1个月速成)**
- **第1-2周：** 搭建 React Native 框架，集成 SQLite，跑通本地的手动录入和搜索闭环。
- **第3周：** 接入智谱 GLM-4V-Flash，联调拍照识别功能，处理断网等异常分支。
- **第4周：** 内部 UI 优化与 Bug 修复，邀请 5-10 名种子用户进行测试反馈。
---
#### **8. 迭代规划 (V1.1 及以后)**
以下功能在 MVP 阶段暂不开发，视用户反馈在后续版本中加入：
- 多端数据同步与云端备份（需引入账号系统）。
- 小票/订单 OCR 批量识别。
- 扫码录入（条形码）。
- 语音对话式查询与录入。
- 国际化（多语言支持）。
